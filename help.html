How the lists work
When the black list is active: if the url matches any of the entries on the list, the tab is muted; otherwise it is unmuted.

When the white list is active: if the url matches any of the entries on the list, the tab is unmuted; otherwise it is muted.

Each line in the list is a separate entry. Each entry can have multiple keywords, separated by spaces. The url must contain all of the keywords for the entry to be considered a match. A keyword can be preceded by a hyphen to indicate that the url must *not* contain the keyword. Matching is case-insensitive.

For example, the following black list



will mute urls like

http://ftp.badsite.com/dumb
https://www.amazon.com/dp/PRODUCT/B002H09I0Y
http://naughtysite.com/videos/naughty.mov

but not

http://www.baddersite.com/
https://www.amazon.com/dp/B00137QS28
http://dirtysite.com/videos/naughty.mov
Other controls
Mute All TabsPanic button. Immediately mutes all sound from the browser.
Unmmute All TabsPauses the extension. Immediately allows sound from all tabs.
Always unmute current tabKeeps the in-focus tab always unmuted, regardless of url.
Always mute all other tabsKeeps all tabs except the in-focus one always muted, regardless of url.
Add/Remove domainAdds/removes the domain name (e.g. "bing.com") of the in-focus tab to/from the shown list.
Add/Remove pageAdds/removes the full url (e.g. "https://www.bing.com/maps") of the in-focus tab to/from the shown list.

"Remove domain/page" will only remove list entries that contain the exact url and have no added keywords or modifications.

You can also override the list for a specific tab by manually un/muting it, and the change will stick until the tab is closed and reopened. To manually un/mute a tab, either 1) right-click the page, 2) use the keyboard shortcut, or 3) enable the handy tab muting control flag in Chrome. If the tab un/mute has been manually changed, a * will appear next to the right-click menu item.

For each tab, the un/mute is decided in the following order:

If "Mute All Tabs" is on, the tab is muted...

otherwise, if "Unmmute All Tabs" is on, the tab is unmuted...

otherwise, if user manually un/muted the tab, that choice stays...

otherwise, if applicable, the tab is un/muted based on "Always unmute current tab" or "Always mute all other tabs"...

otherwise, the tab is un/muted based on the active list.


The extension icon will show a red X if the current tab is muted or a green circle if unmuted.

Privacy statement
This extension requires access to each tab's current url for its main functionality. Because of this, Chrome will warn you that it can potentially "read your browsing history". However, this extension does not record any of the url's you visit. After the url has been used in the decision to mute or unmute the tab, it is discarded from the extension's memory.

To verify this, you can view the extension's source code as follows:

chrome://extensions  →  Turn on developer mode  →  Mute Tabs by Url  →  Inspect views: background page  →  Application  →  Frames  →  top  →  Scripts  →  background.js

click extension icon  →  right-click popup  →  Inspect  →  Application  →  Frames  →  top  →  Scripts  →  popup/help/permissions.js

This extension also requires permission to display notifications when keyboard shortcuts are used, so that users do not accidentally change settings without knowing it.
